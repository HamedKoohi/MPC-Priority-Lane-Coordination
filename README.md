## Detailed Project Description### The Challenge: Combinatorial ExplosionCoordinating lane changes for multiple vehicles on a multi-lane highway is inherently complex. When formulated as a centralized optimization problem, it becomes a mixed-integer problem where the number of possible lane-change combinations grows exponentially with the number of vehicles. This "combinatorial explosion" typically makes real-time implementation impossible for traditional controllers.### The Solution: Priority-Aware CoordinationThis repository implements a novel hierarchical control algorithm that bridges the gap between global optimality and real-time feasibility.The core innovation is a priority-based search strategy integrated into a Model Predictive Control (MPC) framework. Instead of checking every possible combination, the controller:Identifies High-Priority Candidates: It uses a cost-based heuristic to select the three most "promising" vehicles at each time step based on speed errors and surrounding traffic density.Optimizes Jointly: It simultaneously optimizes the longitudinal speed profiles and lane assignments for these prioritized vehicles while maintaining safe following distances for the entire fleet.Ensures Feasibility: By limiting the search space to a subset of vehicles, the computational burden remains constant and manageable, even as the total number of vehicles in the segment increases.### Key Technical ComponentsMPC Framework: Optimizes acceleration, braking, and lane-change commands over a finite prediction horizon in a receding-horizon fashion.Artificial Potential Fields (APFs): Integrated into the cost function to provide continuous safety guidance and proactive collision avoidance.Sigmoid Trajectory Planning: Replaces idealized "instantaneous" lane changes with realistic, finite-duration maneuvers that respect lateral acceleration and jerk constraints for passenger comfort.System-Level Utility: Specifically designed to prevent "deadlocks" by penalizing vehicles that loiter in the left lane when they have lower desired speeds.### Mathematical ObjectiveThe controller minimizes a multi-objective cost function $J$:
$$J = J_{speed} + J_{antilock} + J_{collision} + J_{lane\_stability} + J_{jerk}$$Where:$J_{speed}$ tracks the desired reference velocity for each vehicle.$J_{antilock}$ prevents traffic flow bottlenecks in passing lanes.$J_{collision}$ uses APF-based penalties to maintain safe inter-vehicle gaps.
